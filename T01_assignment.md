## 과제 요약

### 1. 구현 대상 (What to Implement)

이번 과제의 핵심은 MiniC 소스 파일을 읽어 **토큰(Token) 시퀀스**로 변환하는 **Lexical Analyzer(Scanner)**를 직접 손으로(by hand) 구현하는 것입니다.

- **주요 임무**: `Scanner.py` 파일을 수정하여 MiniC 언어 명세에 정의된 모든 캐릭터, 토큰, 주석을 인식할 수 있도록 확장해야 합니다.
- **작동 방식**: 컴파일러 드라이버(`MiniC.py`)가 스캐너를 반복 호출하면, 스캐너는 다음 토큰을 찾아 **Token 객체**를 반환해야 합니다.
- **Maximal Munch 원칙**: 가능한 가장 긴 일치 항목을 하나의 토큰으로 소비해야 합니다. (예: `>`와 `=`가 연달아 있으면 두 개의 토큰이 아닌 하나의 `>=` 토큰으로 인식)
- **제외 항목**: 공백(Whitespace)과 주석(Comments)은 스캔 과정에서 무시하고 버려야 합니다.

### 2. 코드 구조 및 제약 사항

- **수정 가능 파일**: **`Scanner.py` 단 하나만 수정**해야 합니다.
- **수정 금지 파일**: `Token.py`, `SourcePos.py`는 절대 수정해서는 안 됩니다. (수정 시 자동 채점 시스템에서 오류 발생 가능)
- **Token 객체 구성**:
  1.  **Kind**: 토큰의 종류 (예: `Token.ID`, `Token.INTLITERAL`)
  2.  **Lexeme**: 실제 문자열 값 (예: `"counter"`, `"22"`)
  3.  **SourcePos**: 토큰의 시작/끝 라인 및 컬럼 번호 (1부터 시작)

### 3. 에러 처리 방법 (Error Handling)

스캐너는 총 4가지 유형의 렉시컬 에러를 감지하고 처리해야 합니다.

| 에러 유형                 | 처리 방법                                                                                                                                       |
| :------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------- |
| **1) 잘못된 문자**        | `Token.ERROR` 객체를 생성하여 반환합니다. 이때 해당 문자를 소비하여 다음 스캔이 진행될 수 있도록 합니다.                                        |
| **2) 끝나지 않은 주석**   | 파일 끝(EOF)에 도달할 때까지 주석이 닫히지 않으면 에러 메시지를 출력하고 `Token.EOF`를 반환합니다.                                              |
| **3) 끝나지 않은 문자열** | MiniC 문자열은 줄바꿈을 허용하지 않습니다. 닫는 따옴표 없이 줄바꿈을 만나면 에러 메시지를 출력하고, 줄바꿈 전까지의 내용을 토큰으로 반환합니다. |
| **4) 잘못된 이스케이프**  | 문자열 내에 `\n` 이외의 잘못된 이스케이프(예: `\y`)가 있으면 에러 메시지를 출력합니다. 렉세임은 원래 문자열 전체를 포함해야 합니다.             |

> **중요**: 2번~4번 케이스의 에러 메시지에는 반드시 대문자로 **`ERROR:`**라는 문자열이 포함되어야 자동 채점 시스템이 인식할 수 있습니다.
