## 주요 내용 요약

### 재귀 하강 파싱 (Recursive Descent Parsing)

재귀 하강 파싱은 **LL(Top-down) 파싱** 방식의 일종으로, 문법의 각 비단말 기구(Non-terminal)를 하나의 함수 또는 프로시저로 구현하여 구문을 분석하는 방법입니다. 소스 코드의 구조가 호출하는 함수들의 관계와 대응된다는 점이 특징입니다.

### EBNF를 활용한 파서 설계

확장 배커스-나우르 표기법(EBNF)으로 정의된 문법을 기계적인 규칙에 따라 실제 코드로 변환하는 과정을 다룹니다.

- **터미널(Terminal):** `self.accept(token)` 함수를 통해 현재 토큰을 확인하고 다음 토큰으로 넘어갑니다.
- **비단말(Non-terminal):** 해당 기호를 처리하는 `parseN()` 함수를 호출합니다.
- **선택 (X | Y):** 현재 토큰의 종류에 따라 조건문(`if`, `match`)을 사용하여 분기합니다.
- **반복 (X\*):** `while` 루프를 사용하여 해당 패턴이 반복되는 동안 파싱 함수를 계속 호출합니다.

### 문법 변환 (Grammar Transformations)

재귀 하강 파서가 올바르게 동작하기 위해 필수적인 사전 작업입니다.

- **좌인수 분해 (Left Factorization):** 공통된 접두사를 가진 규칙들을 분리하여 파서가 어떤 경로를 선택할지 결정할 수 있게 합니다.
- **좌재귀 제거 (Left Recursion Elimination):** `A -> Aα`와 같이 자기 자신을 먼저 호출하는 규칙은 무한 루프를 유발하므로, 이를 반복문 형태로 변환하여 제거합니다.

### 예외 처리 및 에러 보고

파싱 과정에서 문법 오류가 발생했을 때 이를 탐지하고 보고하는 메커니즘을 설명합니다.

- `ErrorReporter` 클래스를 통해 오류 위치(라인, 컬럼)와 메시지를 출력합니다.
- `SyntaxError` 예외를 발생시켜 파싱 과정을 중단하고 상위 루틴에서 이를 처리하도록 설계합니다.

## 핵심 기술 키워드

### 구문 분석 관련

- **LL Parsing:** 입력을 왼쪽에서 오른쪽으로 읽고(Left-to-right), 가장 왼쪽 유도(Leftmost derivation)를 수행하는 하향식 파싱 방식입니다.
- **First Set:** 특정 문법 규칙에서 유도될 수 있는 첫 번째 터미널 기호들의 집합입니다. 파서가 어떤 함수를 호출할지 결정하는 기준이 됩니다.
- **Recursive Descent:** 문법 규칙을 재귀적인 함수 호출로 직접 구현한 파싱 알고리즘입니다.

### 문법 구조 관련

- **EBNF (Extended Backus-Naur Form):** 프로그래밍 언어의 문법을 정의하는 메타 언어입니다. `*`, `|`, `()` 등을 사용하여 반복과 선택을 표현합니다.
- **Common Prefix:** 여러 규칙이 동일한 시작 토큰을 공유하는 현상으로, 좌인수 분해를 통해 해결해야 합니다.
- **Left Recursion:** 비단말 기호가 정의의 가장 왼쪽에 자기 자신을 포함하는 경우로, 하향식 파서에서는 반드시 제거해야 합니다.

### 구현 클래스 및 메서드

- **Scanner:** 소스 코드를 토큰 스트림으로 변환하는 역할을 하며, 파서의 `accept` 메서드에 의해 호출됩니다.
- **accept(expected):** 현재 토큰이 기대하는 토큰과 일치하는지 검사하고 다음 토큰을 읽어오는 핵심 보조 메서드입니다.
- **parseProgram / parseExpr:** 특정 문법 단위(프로그램, 식 등)를 파싱하기 위해 구현된 개별 프로시저입니다.

### 오류 관리

- **ErrorReporter:** 컴파일 과정에서 발견된 오류의 개수를 카운트하고 사용자에게 상세 위치를 알리는 객체입니다.
- **Panic Mode Recovery (Implicit):** 자료에서는 예외(Exception)를 던져 즉시 중단하는 형태의 기초적인 오류 처리 방식을 보여줍니다.
