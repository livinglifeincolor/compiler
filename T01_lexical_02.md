## 키워드 정리

### 1. NFA에서 DFA로의 변환 (NFA → DFA)

NFA의 비결정론적 특성을 제거하여 컴퓨터가 효율적으로 처리할 수 있는 DFA로 바꾸는 과정입니다.

- **Subset Construction (부분집합 구축)**: NFA의 상태들의 집합을 하나의 새로운 DFA 상태로 만드는 알고리즘입니다.

- **Virtual State (가상 상태)**: NFA에서 동시에 존재할 수 있는 여러 상태의 집합을 하나의 상태로 묶은 것입니다.

- **$\epsilon$-closure (이동)**: 입력 문자 소비 없이 $\epsilon$(빈 문자열) 전이만으로 도달할 수 있는 모든 상태의 집합을 구하는 함수입니다.

- **Move(S, a)**: 특정 상태 집합(S)에서 입력 기호 'a'를 받았을 때 도달 가능한 모든 상태 집합을 반환하는 함수입니다.

### 2. DFA 상태 최소화 (DFA State Minimization)

DFA의 상태 수 중 불필요한 것을 줄여 가장 효율적인 'Minimal DFA'를 만드는 과정입니다.

- **Distinguishable vs Indistinguishable**: 두 상태에 같은 입력 문자열을 주었을 때, 하나는 수락 상태로 가고 다른 하나는 거부 상태로 가면 '구분 가능(Distinguishable)'하다고 하며, 결과가 같으면 '구분 불가능(Indistinguishable)'하다고 합니다.

- **Partitioning (파티셔닝)**: 전체 상태를 구분 가능한 그룹으로 나누는 과정입니다. 처음에는 '수락 상태 집합(F)'과 '비수락 상태 집합(S-F)' 두 그룹으로 시작합니다.

- **Split (분할)**: 같은 그룹 내의 상태들이 특정 입력 기호에 대해 서로 다른 그룹으로 이동할 경우, 해당 그룹을 더 작은 단위로 쪼개는 과정입니다.

- **Fixed-point Algorithm**: 더 이상 파티션이 쪼개지지 않을 때까지 반복하며 최종 해에 도달하는 방식의 알고리즘입니다.

### 3. 스캐너 구현 및 자동화

- **Scanner Generator**: 정규 표현식(RE)을 입력하면 위 알고리즘들을 거쳐 자동으로 Minimal DFA 기반의 스캐너 코드를 생성해주는 도구(예: Lex, Flex)입니다.

- **The Cycle of Constructions**: **RE → NFA → DFA → Minimal DFA**로 이어지는 컴파일러 프론트엔드의 표준 자동화 프로세스입니다.

## Subset Construction(부분집합 구축) 알고리즘: NFA를 DFA로 변환하는 과정

### 1. 핵심 개념: Virtual State (가상 상태)

NFA는 하나의 입력에 대해 여러 상태로 갈 수 있는 '비결정론적' 특성이 있습니다. 이를 DFA로 바꾸기 위해, **"NFA에서 동시에 존재할 수 있는 상태들의 집합"**을 하나의 새로운 **가상 상태(Virtual State)**로 간주하는 것이 이 알고리즘의 핵심입니다.

### 2. 주요 함수 (수업 자료 정의)

알고리즘을 수행하기 위해 두 가지 핵심 함수를 사용합니다.

- **$\epsilon$-closure(S)**: 상태 집합 $S$에 속한 각 상태로부터 입력 없이($\epsilon$ 이동) 도달할 수 있는 모든 상태의 집합을 구합니다.
- **Move(S, a)**: 상태 집합 $S$에 속한 상태들에서 입력 기호 'a'를 얻었을 때 도달할 수 있는 NFA 상태들의 집합을 구합니다.

### 3. 단계별 변환 과정

수업 자료의 알고리즘 흐름을 정리하면 다음과 같습니다.

**1단계: DFA의 시작 상태 생성**

- NFA의 시작 상태를 $s_0$라고 할 때, DFA의 시작 상태 $q_0$는 **$\epsilon$-closure($s_0$)**가 됩니다.
- 이 $q_0$를 '아직 처리되지 않은 상태' 목록에 넣습니다.

**2단계: 반복적인 상태 확장 (Loop)**
'처리되지 않은 상태'가 없을 때까지 다음 과정을 반복합니다.

1. 목록에서 DFA 상태 $T$를 하나 꺼냅니다.
2. 알파벳 세트($\Sigma$)에 있는 각 입력 기호 $a$에 대해 다음을 계산합니다.

- **$U = \epsilon\text{-closure(Move}(T, a))$**

3. 만약 이 새로운 집합 $U$가 이전에 발견되지 않은 새로운 상태라면:

- $U$를 DFA의 새로운 상태로 추가하고 '처리되지 않은 상태' 목록에 넣습니다.

4. DFA의 전이 함수를 설정합니다: **$Dtran[T, a] = U$**.

**3단계: 종료 상태(Accepting State) 지정**

- 생성된 DFA의 상태(집합)들 중, **원래 NFA의 종료 상태를 하나라도 포함하고 있는 집합**들을 모두 DFA의 종료 상태로 지정합니다.

### 4. 실제 예시 이해하기

만약 NFA에서 상태 $S_1$이 'a'를 읽고 $S_1$과 $S_2$ 두 곳으로 갈 수 있다면:

- DFA에서는 **$\{S_1, S_2\}$**라는 이름의 하나의 거대한 상태를 만듭니다.
- 컴퓨터는 이제 "어디로 갈지 추측"할 필요 없이, 단순히 ${S_1, S_2}$라는 상태에 있다고 판단하면 되므로 동작이 결정론적(Deterministic)으로 변하게 됩니다.

## Instead of guessing in state s1 on symbol a, we can follow both transitions (s1→s1 and s1→s2) in parallel.

제공해주신 강의 자료에 따르면, 해당 문장은 **NFA(비결정적 유한 오토마타)**의 동작 특성과 이를 **DFA(결정적 유한 오토마타)**로 변환하는 **Subset Construction(부분집합 구성법)**의 핵심 원리를 설명하고 있습니다.

기술적인 관점에서 상세히 설명해 드리겠습니다.

### 1. NFA에서의 'Guessing' (추측) 문제

NFA는 동일한 상태에서 동일한 입력 심볼을 받았을 때 갈 수 있는 경로가 여러 개인 경우가 존재합니다.

- **상태 $S_{1}$의 사례**: 그림을 보면 상태 $S_{1}$에서 문자 `a`를 읽었을 때, 자기 자신인 $S_{1}$에 머물 수도 있고 다음 상태인 $S_{2}$로 넘어갈 수도 있습니다.

- **추측의 필요성**: 이 시점에서 오토마타는 "나중에 문자 `b`가 연달아 두 번 나와서 최종 상태 $S_{4}$에 도달하려면 지금 $S_{2}$로 가야 할까, 아니면 그냥 $S_{1}$에 있어야 할까?"를 **추측(Guess)**해야 합니다.

- **실패 가능성**: 만약 추측을 잘못하여 잘못된 경로를 선택하면, 실제로는 해당 언어에 포함되는 유효한 문자열임에도 불구하고 거부(Reject)될 수 있습니다.

### 2. 'Follow in Parallel' (병렬 추적)의 의미

컴퓨터가 실제로 실행될 때는 "추측"을 할 수 없으므로, 가능한 모든 경로를 **동시에(In parallel)** 탐색하는 방식을 취합니다.

- **가상 상태(Virtual State) 도입**: 문자 `a`를 읽었을 때 도달 가능한 모든 상태들의 집합을 하나의 새로운 상태로 정의합니다.

- **집합적 관리**: $S_{1}$에서 `a`를 읽고 갈 수 있는 곳이 ${S_{1}, S_{2}}$라면, 이를 하나의 **"상태 집합"**으로 묶어 현재 오토마타가 이 두 상태에 동시에 존재한다고 간주합니다.

- **DFA로의 변환**: 이렇게 모든 가능한 상태의 조합을 새로운 하나의 상태로 만드는 과정이 바로 **Subset Construction(NFA → DFA)** 알고리즘의 본질입니다.

### 3. 요약

즉, 질문하신 문장은 **"NFA처럼 어느 길로 갈지 고민하며 도박을 하는 대신, 가능한 모든 경로를 하나의 집합으로 묶어 관리함으로써 결정론적(Deterministic)으로 동작하게 만들 수 있다"**는 뜻입니다.

구체적으로는 다음 두 함수를 통해 이를 구현합니다:

1.  **Move($S_{i}$, a)**: 상태 집합 $S_{i}$에서 입력 `a`를 통해 갈 수 있는 모든 상태를 찾음.

2.  **$\epsilon$-closure($S_{i}$)**: $\epsilon$ 이동(입력 없이 이동)을 통해 갈 수 있는 모든 상태까지 포함함.

이 과정을 반복하여 더 이상 새로운 상태 집합이 생기지 않을 때까지 DFA를 구축하게 됩니다.

업로드해주신 이미지는 **NFA(비결정적 유한 오토마타)**를 **DFA(결정적 유한 오토마타)**로 변환하는 핵심 알고리즘인 **Subset Construction(부분집합 구성법)**의 절차를 기술하고 있습니다.

NFA는 하나의 입력에 대해 여러 상태로 갈 수 있는 비결정론적 특성을 가지지만, 컴퓨터는 한 번에 하나의 상태만 가질 수 있으므로 이를 시뮬레이션하기 위해 가능한 상태들의 **부분집합(Subset)**을 하나의 상태로 묶어 관리하는 것이 이 알고리즘의 핵심입니다.

주요 구성 요소와 단계별 과정은 다음과 같습니다.

### 1. 두 가지 핵심 함수 (Two Key Functions)

알고리즘을 수행하기 위해 정의된 연산입니다.

- **Move($s_{i}$, a)**: 상태 집합 $s_{i}$에 속한 상태들에서 입력 기호 `a`를 통해 도달할 수 있는 모든 NFA 상태들의 집합을 반환합니다.

- **$\epsilon$-closure($s_{i}$)**: 상태 집합 $s_{i}$에서 입력 기호 없이 $\epsilon$ (null) 이동만으로 도달할 수 있는 모든 상태들의 집합을 반환합니다.

### 2. 알고리즘 수행 단계 (The Algorithm)

DFA의 상태들을 하나씩 찾아가는 과정입니다.

1.  **시작 상태 설정**: NFA의 시작 상태인 $s_{0}$에 대해 $\epsilon$-closure를 수행하여 DFA의 시작 상태 $S_{0}$를 생성합니다 ($S_{0} = \epsilon\text{-closure}(\{s_{0}\})$).

2.  **상태 전이 계산**: 현재 찾은 DFA 상태(상태 집합) $S_{0}$에 대해 모든 입력 기호 $\alpha \in \Sigma$에 대해 전이를 계산합니다.

- 먼저 `Move(S0, α)`를 계산하고, 그 결과에 다시 `ε-closure`를 적용하여 새로운 DFA 상태를 도달합니다.

3.  **반복**: 더 이상 새로운 DFA 상태가 추가되지 않을 때까지 모든 상태와 입력 기호 조합에 대해 이 과정을 반복합니다.

### 요약하자면

이 알고리즘은 NFA에서 발생할 수 있는 모든 상태의 조합을 각각의 새로운 **DFA 상태**로 정의함으로써, 비결정론적인 움직임을 결정론적(Deterministic)으로 변환합니다. 결과적으로 생성된 DFA의 특정 상태가 NFA의 수락 상태(Accepting state) 중 하나라도 포함하고 있다면, 그 DFA 상태는 수락 상태가 됩니다.

## DFA Minimization

DFA 최소화(DFA Minimization)는 동일한 언어를 인식하면서도 상태(State)의 개수가 가장 적은 최적의 결정적 유한 오토마타를 만드는 과정입니다. 핵심 아이디어는 **"결과가 같은 상태들은 하나로 합쳐도 무방하다"**는 점에 기반하며, 이를 위해 **분할(Partitioning) 알고리즘**을 사용합니다.

### 1. 핵심 개념: 구별 가능성 (Distinguishability)

두 상태 $p$와 $q$가 있을 때, 어떤 입력 문자열 $w$를 넣었을 때 하나는 수락(Accept) 상태로 가고 다른 하나는 비수락 상태로 간다면, 두 상태는 **구별 가능(Distinguishable)**하다고 합니다. 반대로 모든 가능한 입력에 대해 결과가 같다면 두 상태는 **구별 불가능(Indistinguishable)**하며, 이들은 하나의 상태로 합칠 수 있습니다.

### 2. 알고리즘 작동 단계 (Fixed-point Computation)

이 알고리즘은 더 이상 상태가 나누어지지 않을 때까지 반복하는 **고정점 계산(Fixed-point computation)** 방식을 따릅니다.

#### **Step 1: 초기 분할 (Initial Partition)**

먼저 전체 상태 집합 $S$를 수락 상태 그룹($F$)과 비수락 상태 그룹($S-F$)의 두 파티션으로 나눕니다.

- **예**: $P_0 = \{ \{s_4\}, \{s_0, s_1, s_2, s_3\} \}$ (여기서 $s_4$는 수락 상태)

#### **Step 2: 반복적 분할 (Splitting)**

각 파티션 내의 상태들이 특정 입력 $\alpha$에 대해 **서로 다른 파티션**으로 전이되는지 확인합니다.

- 만약 어떤 파티션 $\{s_a, s_b\}$에서 $s_a$는 입력 'a'를 받고 그룹 A로 가는데, $s_b$는 입력 'a'를 받고 그룹 B로 간다면, $s_a$와 $s_b$는 서로 다른 동작을 하는 것이므로 분리(Split)해야 합니다.

##### **`Split(p)` 함수의 동작 원리**

이 함수는 현재 검사 중인 특정 파티션 $p$ 내부의 상태들이 **동질성(Homogeneity)**을 유지하는지 확인합니다.

- **입력 기호 순회:** 알파벳 집합 $\Sigma$에 속한 각 문자 $a$에 대해 순차적으로 검사합니다.
- **전이 결과 확인:** 파티션 $p$ 내의 각 상태에 문자 $a$를 입력했을 때 도달하는 상태들이 **현재 어느 파티션에 속해 있는지**를 봅니다.
- **분리 조건:** 만약 상태들이 전이된 결과값들이 **서로 다른 파티션($p_i$와 $p_k$ 등)**으로 갈라진다면, 이는 해당 상태들이 현재 파티션 $p$ 내에서 더 이상 함께 있을 수 없음을 의미합니다.
- **분리 실행:** 이 조건이 만족되면 $p$를 $p1$과 $p2$로 분할하여 반환합니다.

#### **Step 3: 종료 (Termination)**

더 이상 파티션이 나누어지지(Split) 않는 상태에 도달하면 알고리즘을 종료합니다. 이때 남은 각 파티션이 최소화된 DFA의 새로운 개별 상태가 됩니다.

### 3. 왜 이 과정이 중요한가요?

- **정합성 유지**: 최소화된 DFA는 원래의 DFA와 기술적으로 동일한 정규 언어를 인식합니다.
- **효율성**: 상태 수가 줄어들면 메모리 사용량이 감소하고, 스캐너(Scanner)의 동작 속도가 최적화됩니다.
- **유일성**: 이론적으로 모든 정규 언어는 상태 이름만 다를 뿐, 유일한 최소 상태 DFA를 가집니다.
